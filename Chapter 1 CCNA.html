IPv4 Addresses

What are IPv4 addresses used for?

  • Created to allow one specific digital device to communicate directly with another specific digital device
    • This is called unicast communication
  • Needs to allow for one device to access multiple devices

An IPv4 address is known as a hierarchical address because it can be divided into multiple smaller domains.

An IPv4 address in conjunction with a subnet gives you two pieces of information:

  • The network in which the device is located
  • The unique identifier of the device
Note: Different from a MAC address—a MAC address is a flat address. It does not give information about the network it's connected to.

What does an IPv4 address look like?

  • The computer sees a 32-bit number
  • For us they are broken up into 4 groups of 8 bits called bytes or octets, separated by periods
  • For example:
    • 11000000.10101000.00000001.00000001
    • This would result in 192.168.1.1

More about binary (the type of language this is)

  • You start with 00000000
  • The first bit stands for 1
  • The second bit stands for 2
  • The third bit stands for 4
  • The fourth bit stands for 8
  • The fifth bit stands for 16
  • The sixth bit stands for 32
  • The seventh bit stands for 64
  • The eighth bit stands for 128
  • This is assuming it is an 8-bit number; anything past is the same pattern (just keep doubling until you reach the number)
Examples:
  • 00011111 is 31 (it would not be 32 because 00100000 is 32)
  • 11111111 is 255, the largest possible number for an 8-bit number
  • For practice: what is 11010011? — (answer: 227)

IPv4 needs some way to differentiate the network portion from the host portion. To do this, all IPv4 addresses have a network mask (or subnetwork mask, subnet mask, or mask).

A network mask is 32 bits that define what portion represents the network and which portion is the host.

Rules for network masks

  • A 1 in the mask defines a network bit
  • A 0 in the mask defines a host bit

3 ways to denote a subnet mask

  1. Binary: 11111111.11111111.11111111.00000000
  2. Decimal: 255.255.255.0
  3. Slash notation: /24

All of them tell us the first 24 bits of the 32-bit mask are 1 and the last 8 are 0.

Compared to an IPv4 address, the first 24 bits are the network portion of the address and the last 8 bits define the host.

If you add them you get a result like this:
  • 11000000.10101000.00000001.00000001 (address)
  • 11111111.11111111.11111111.00000000 (mask)
  • 192.168.1.1 255.255.255.0
  • 192.168.1.1/24

First designed to allow for people to recognize whether an IP address connected to a specific device or a group of devices.

In an IP address:

  • A host portion of all binary 0's refers to the network itself
  • A host portion with a combination is a specific device
  • A host portion of all binary 1's refers to the broadcast address of the network
Using 192.168.1.1/24 for our example:
  • All 0's in the host = the network itself → 192.168.1.0
  • Host portion 00000001 = first host → 192.168.1.1
    • Ranges up to 11111110, giving 254 unique addresses: 192.168.1.1–192.168.1.254
  • Host portion of all 1's = broadcast address → 192.168.1.255

Classes of IPv4 Addresses

IPv4 addresses are divided into 5 different classes according to size.

Note: Not currently used due to Classless Inter-Domain Routing (CIDR), but still good to know.

Broken down by a concept called "leading bit pattern" of the first octet of an IPv4 address.

Class Leading Bit Pattern First Octet (Decimal) Notes
A 0xxxxxxx 0–127 0 is invalid; 10.0.0.0/8 reserved for private internal routing; 127 reserved for loopback testing
B 10xxxxxx 128–191 172.16.0.0/12 reserved for private internal routing
C 110xxxxx 192–223 192.168.0.0/16 reserved for private internal routing
D 1110xxxx 224–239 Reserved for multicasting; cannot be assigned to unicast hosts
E 1111xxxx 240–255 Reserved for future use/testing
  • Classes A, B, and C are the only classes able to use unicast communication
  • Class D is for multicast communication
  • Class E will never be released for unicast communication

Network vs. Node (Host) Bits

Within classes A to C, the four octets were broken down into either network octets (N) or node/host octets (H).

Class Octet 1 Octet 2 Octet 3 Octet 4 Default Network Mask
A NNNNNNNN HHHHHHHH HHHHHHHH HHHHHHHH /8 or 255.0.0.0
B NNNNNNNN NNNNNNNN HHHHHHHH HHHHHHHH /16 or 255.255.0.0
C NNNNNNNN NNNNNNNN NNNNNNNN HHHHHHHH /24 or 255.255.255.0
  • Class A has 24 bits for hosts. 224 = 16,777,216. Removing two (network ID + broadcast) leaves 16,777,214 available hosts per network.
  • Class B has 16 bits for hosts. 216 = 65,536. Subtracting two leaves 65,534 valid hosts per network.
  • Class C has 8 bits for hosts. 28 = 256. Subtracting two leaves 254 valid hosts per network.

Conclusions

  • There are 126 Class A networks of 16,777,214 valid hosts each
  • There are 16,384 Class B networks of 65,534 valid hosts each
    • From the first 16 bits, but two are fixed (10). That leaves 14 bits → 214 = 16,384
    • Ranges from 10000000.00000000 (128.0) to 10111111.11111111 (191.255)
  • There are 2,097,152 Class C networks of 254 valid hosts each
    • First 24 bits, with first three fixed (110). That leaves 21 bits → 221 = 2,097,152
    • Ranges from 11000000.00000000.00000000 (192.0.0) to 11011111.11111111.11111111 (223.255.255)
Example: Using this information, 192.168.1.1/24 is a Class C network.
