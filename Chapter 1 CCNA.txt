--IPv4 Addresses--

-What are IPv4 addresses used for?-
	- Created to allow on specific digital device to communicate directly with another specific digital device
		- This is called unicast communication
	- Needs to allow for one device to access multiple device
	
IPv4 address is known as a hierarchical address because it can be divided into multiple smaller domains 
An IPv4 address in conjunction with a subnet gives you two pieces of infomation
	- The network in which the device is loaded
	- The unique identifier of the device
(Different from a MAC address, MAC address is a flat address. It does not give infomation about the network it's connected to.)


-What does a IPv4 address look like?-
	- The computer sees a 32-bit number 
	- For us they are broken up into 4 groups of 8 bits called bytes or octet. Seperated by periods
	- For example:
		- 11000000.10101000.00000001.00000001
		- This would result in 192.168.1.1
		
		More about binary (the type of language this is)
		- You start with 00000000
		- The first 0 stands for 1
		- The second 0 stands for 2
		- The third 0 stands for 4
		- The forth 0 stands for 8
		- The fifth 0 stands for 16
		- The sixth 0 stands for 32
		- The seventh 0 stands for 64
		- The eighth 0 stands for 128
		- This is assumming it is a 8 bit number, anything past is the same pattern (just keep doubling until you reach the number)
		- For example:
			- 00001111 is 31, it would not be 32 because 00010000 is 32
			- 11111111 is 255, the largest possible number for a 8 bit number
			- For practice, what is 11010011 ----------------------------- (answer is 227)
			
IPv4 needs some way to differenate the network portion to the host portion. To do this all IPv4 address have a network mask (or subnetwork mask, subnet mask or mask)
A network mask is 32 bits that define what portion reprents the network and which portion is the host. 
Rules for network masks
	- A 1 in the mask defines a network bit
	- A 0 in the mask defines a host bit
	
3 ways to donote a subnet mask:
	- Binary
		- 11111111.11111111.11111111.00000000
	- Decimal
		- 255.255.255.0
	- Slash Notation
		- /24
		
	All of them tell us the first 24 bits of the 32 bit mask at 1 and the last 8 are 0
	Compared to a IPv4 Address the first 24 bits are the network portion of the address and the last 8 bits define the host
	If you add them you get a result like this:
		- 11000000.10101000.00000001.00000001 (address)
		- 11111111.11111111.11111111.00000000 (mask)
		- 192.168.1.1 255.255.255.0
		- 192.168.1.1/24 
	
	First designed to allow for people to recognize whether an IP address connected to a specific device or a group of devices
		In a IP address:
			- A host portion of all binary 0's refers to the network itself
			- A host portion with a combonation is a specific device
			- A host portion with all binary 1's refers to the broadcast address of the network
			
			- We will use 192.168.1.1/24 for everything 
				- All 0's in the host means it's the network itself 
					- 192.168.1.0
				- A host portion with a binary of 00000001 is the first host of the network
					- 192.168.1.1
					- This also ranges up to 11111110 give a range of 254 unique addresses
						- 192.168.1.1-192.168.1.254
				- A host portion of all binary 1's means it is the broadcast address
					- 192.168.1.255
					
Classes of IPv4 Addresses

IPv4 addresses divided into 5 different classes according to size.
(Not currently used due to Classless interdomain routing but still good to know)

Broken down by a concept called "leading bit pattern" of the first octet of a IPv4 address

Below is the class specifications:

Class	|Leading Bit Pattern	|First Octet (Decimal)	|Notes
A		|0xxxxxxx				|0-127					|0 is invalid, 10.0.0.0/8 is reserved for private, internal routing only, 127 is reserved for loopback testing
B		|10xxxxxx				|128-191				|127.16.0.0/12 is reserved for private, internal routing only
C 		|110xxxxx				|192-223				|192.168.0.0/16 is reserved for private, internal routing only
D 		|1110xxxx				|224-239				|Reserved for multicasting; cannot be assigned to unicast hosts
E		|1111xxxx				|240-255				|Reserved for future use\testing

Class A, B and C are the only classes able to use unicast communication
Class D is for multicast communication
Class E will never be released for unicast communication


Network vs. Node (Host) bits

Within class A to C, the four octets were broken down to either network octets or node octets. 

Class	|Octet 1		|Octet 2		|Octet 3		|Octet 4		|Default network mask
A 		|NNNNNNNN		|HHHHHHHH		|HHHHHHHH		|HHHHHHHH		|/8 or 255.0.0.0
B 		|NNNNNNNN		|NNNNNNNN		|HHHHHHHH		|HHHHHHHH		|/16 or 255.255.0.0
C 		|NNNNNNNN		|NNNNNNNN		|NNNNNNNN		|HHHHHHHH		|/24 or 255.255.255.0

Class A has 24 bits to assign hosts. 2^24 = 16,777,216. Removing two hosts for network ID (all 0 for host portion, all 1 for broadcast portion.)
This leaves 16,777,214 avaible for in a single network. 

Class B has 16 bits to assign hosts. 2^16 = 65,536. 
Subtracting the same two hosts leaves 65,534 vaild hosts per network. 

Class C has 8 bits to assign hosts. 2^8 = 256. 
Subtracting the same two hosts leaves 254 vaild hosts per network. 

Following this infomation, we can make these conclusions. 
	- There are 126 Class A networks of 16,777,214 vaild hosts each
	- There are 16,384 Class B networks of 65,534 valid hosts each
		- We get this from the first 16 bits, but the two of them are fixed in the pattern of 10.
			- This means that we have 14 bits left to assign addresses. 
			- This ranges from 10000000.00000000 (128.0) to 10111111.11111111 (191.255)
			- This results in 16,384 differnt combonations/networks
	- There are 2,097,152 Class C networks of 254 valid hosts each.
		- We get this from the first 24 bits, the first three of them are fixed in the pattern of 110.
			- That means we have 21 bits left to assign addresses. 
			- This ranges from 11000000.00000000.00000000 (192.0.0) to 11011111.11111111.11111111 (223.255.255)
			- That results in 2,097,152 differnt combonations/networks
			
Using this infomation, we can tell that 192.168.1.1/24 is a class C network
